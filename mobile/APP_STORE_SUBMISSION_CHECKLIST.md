# App Store Submission Checklist

Use this checklist to prepare your NeoSync app for submission to the App Store.

## Required Information

- [ ] **App Name**: NeoSync
- [ ] **App Icon**: 1024x1024 PNG (located in mobile/assets/icon.png)
- [ ] **App Description**: Compelling description of your app (see template below)
- [ ] **Keywords**: List of relevant keywords for App Store search
- [ ] **Support URL**: Website where users can get support
- [ ] **Privacy Policy URL**: Link to your privacy policy
- [ ] **Marketing URL** (optional): Website for marketing your app
- [ ] **App Store Screenshots**: For various device sizes (6.5", 5.5", 4.7", 4.0", iPad, etc.)
- [ ] **App Store Preview** (optional): Short video showcasing your app

## App Description Template

```
NeoSync: Quantum-Secure Messaging

NeoSync is a cutting-edge messaging app that puts your privacy first. With advanced end-to-end encryption, your conversations stay completely private.

Key Features:
• Quantum-secure encryption technology

Your data belongs to you and only you. Unlike other messaging apps, NeoSync never sells your information to advertisers or scans your messages for data collection.

Download NeoSync today and experience truly private communication.
```

## Screenshots Required

You need screenshots for each device type:

- 6.5" Super Retina Display: 1284 x 2778 pixels (iPhone 13 Pro Max, 12 Pro Max)
- 5.5" Display: 1242 x 2208 pixels (iPhone 8 Plus)
- 4.7" Display: 750 x 1334 pixels (iPhone SE 2nd generation)
- iPad Pro: 2732 x 2048 pixels (12.9-inch iPad Pro)

Recommended screenshots to include:
1. Login/Welcome screen
2. Conversations list
3. Individual conversation with messages
4. Group chat
5. Profile view
6. Settings/preferences

## Technical Requirements

- [ ] **App Binary**: Build with the "iOS App Store" distribution method
- [ ] **App Version**: Must be higher than any previous version
- [ ] **Build Number**: Must be higher than any previous build
- [ ] **Device Compatibility**: Specify minimum iOS version
- [ ] **App Capabilities**: Ensure all required entitlements are configured
- [ ] **In-App Purchases**: Configure if applicable
- [ ] **App Privacy**: Complete Apple's privacy questionnaire

## Pre-submission Testing

- [ ] Test app launch and onboarding
- [ ] Test all main user journeys
- [ ] Test on multiple device sizes
- [ ] Test with slow network connection
- [ ] Test background app behavior
- [ ] Test notifications
- [ ] Test app permissions

## Common Rejection Reasons

- Incomplete information in the App Store listing
- Crashes and bugs
- Inaccurate description
- Complex or confusing user interface
- Broken links
- Placeholder content
- Not enough functionality or value
- Privacy concerns or missing privacy policy
- Similarity to other apps

## Submission Process

1. Create your app in App Store Connect
2. Complete all metadata fields
3. Upload screenshots and preview
4. Upload your build using EAS Submit
5. Complete the App Review Information section
6. Submit for review

After submission, Apple typically takes 1-3 days to review your app, although this can vary.

## App Store Guidelines

Review the [App Store Review Guidelines](https://developer.apple.com/app-store/review/guidelines/) to ensure your app complies with Apple's requirements.
